## 2.45.0 (June 28, 2026)

- **Feature: journal index relocated from cached system prompt to `<live-context>`** (`src/zrb/llm/prompt/live_context.py`, `src/zrb/llm/prompt/manager.py`, `src/zrb/llm/summarizer/history_summarizer.py`, `src/zrb/llm/task/llm_task.py`, `src/zrb/llm/agent/subagent/manager/manager.py`, `src/zrb/llm/prompt/markdown/journal_mandate.md`, ADR-0082): The journal index snapshot now lives in the per-turn `<live-context>` block instead of the cached system prompt, so provider prompt caching survives across journal writes (the common case). Injected at exactly two moments ‚Äî session start and each summarization ‚Äî and only when `journal_mandate` is an active section. Sub-agents inherit the same coupling. Net effect: the system prompt prefix is now byte-stable for a full session.

- **Feature: model-adaptive prompt profiles** (`src/zrb/llm/prompt/profile.py`, `src/zrb/llm/prompt/manager.py`, `src/zrb/llm/prompt/prompt.py`, `src/zrb/llm/prompt/markdown/persona.explicit.md`, `src/zrb/llm/prompt/markdown/examples.md`, `src/zrb/llm/prompt/markdown/examples.explicit.md`, `src/zrb/config/mixins/llm_prompt.py`, `docs/configuration/llm-config.md`, ADR-0083): Add a second axis to prompt composition ‚Äî per-section phrasing variants selected by `ZRB_LLM_PROFILE` (`terse`/`explicit`/`auto`, default `auto`). `auto` resolves to `terse` unless a per-model profile has been **declared** via `register_model_profile(pattern, profile)`, which mirrors the existing `model_capabilities` registry (no guessing from the model id). The `explicit` profile ships the first variant pair (`persona.explicit.md`) and a few-shot `examples` component that toggles on for `explicit` models. Existing setups are a strict no-op: the default `auto` with no declarations ‚Üí `terse`.

- **Feature: `secret` flag for `EnvField`** (`src/zrb/config/env_field.py`, `src/zrb/builtin/config.py`): New `secret=True` parameter suppresses sensitive config values in `zrb config explain` ‚Äî displayed as `[set]`/`[unset]` instead of the raw value. Applied to `LLM_API_KEY`, `BRAVE_API_KEY`, `SERPAPI_KEY`, `WEB_SUPER_ADMIN_PASSWORD`, and `WEB_SECRET_KEY`.

- **Improvement: conditional tool registration** (`src/zrb/llm/common_tools.py`): Worktree tools (`EnterWorktree`, `ExitWorktree`, `ListWorktrees`) now register only inside a git repository; plan-mode and `AskUserQuestion` tools register only in interactive sessions. Reduces prompt weight in non-interactive/sub-agent contexts where these tools are dead weight.

- **Improvement: user-facing env var references** (`src/zrb/config/mixins/llm_voice.py`, `src/zrb/llm/agent/gates.py`, `src/zrb/llm/tool/shell.py`, `src/zrb/llm/ui/base/commands_mixin.py`): Hardcoded `ZRB_` prefixes in `EnvField` docstrings and user-facing error/system-suggestion messages replaced with `{ENV_PREFIX}` or dynamic lookup via `CFG.ENV_PREFIX`, so white-label users see their own prefix.

- **Improvement: expanded `EnvField` documentation** (`src/zrb/config/mixins/foundation.py`, `src/zrb/config/mixins/hooks.py`, `src/zrb/config/mixins/internet_search.py`, `src/zrb/config/mixins/llm_core.py`, `src/zrb/config/mixins/llm_sandbox.py`): Multi-line structured docstrings with bullet-point value lists for `LOGGING_LEVEL`, `HOOKS_LOG_LEVEL`, `SEARCH_INTERNET_METHOD`, `LLM_PERMISSION_RULES`, `LLM_SANDBOX_OS_SHELL`, `LLM_SANDBOX_FALLBACK`, and `SEARXNG_SAFE`.

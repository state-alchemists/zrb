## 2.48.4 (July 4, 2026)

- **Fix: task readiness/retry race conditions** (`src/zrb/task/base/execution.py`, `src/zrb/task_status/task_status.py`): `mark_as_started()` now runs before the action coroutine's first suspension point, closing a window where two upstreams completing in the same event-loop tick could both pass `is_allowed_to_run` and run the action twice. Readiness now gates on `is_permanently_failed` instead of the per-attempt `is_failed`, which used to race with the retry loop and could skip `mark_as_ready` forever. A readiness check that exhausts its own retries now cancels the action, fails the task, and runs fallbacks immediately instead of leaving the run hanging in `wait_deferred` with no error exit. `allow_run_downstream` (`task_status.py`) has the same `is_permanently_failed`-only fix.
- **Fix: cancellation swallowed as a normal completion** (`src/zrb/task/base/lifecycle.py`, `src/zrb/task/base/monitoring.py`, `src/zrb/callback/callback.py`, `src/zrb/task/base_task.py`): `execute_root_tasks` now re-raises `CancelledError`/`KeyboardInterrupt` instead of returning `None`, so a cancelled session no longer looks like a successful run to its caller. `_handle_threshold_reached` distinguishes an expected action-cancel from the monitor itself being cancelled (via `Task.cancelling()`) instead of swallowing both. `Callback.async_run` and `BaseTask` no longer publish/report cancellation as a task failure.
- **Fix: cron parsing edge cases** (`src/zrb/util/cron.py`): Wildcard steps inside a comma-separated list (`1,*/5`) now parse correctly; out-of-range field values (e.g. minute `70`) now raise `ValueError` instead of silently producing a schedule that never fires; day-of-week accepts `7` as an alias for Sunday.
- **Fix: `Scheduler` cron ticks** (`src/zrb/task/scheduler.py`): Dedupes on the fired minute so a sub-minute tick interval no longer fires a matching minute more than once, sleeps are now clamped to the next minute boundary so drift can no longer cause a whole minute (and its scheduled fire) to be skipped, and the loop now exits once the session is terminated.
- **Fix: approval channel race denies too early** (`src/zrb/llm/approval/multiplex_approval_channel.py`): A channel erroring out no longer resolves the shared future with a denial by itself â€” that could deny a request before a slower channel (e.g. a human on the terminal) got a chance to answer. Denial now only fires once every channel has finished without producing a result.
- **Fix: chat session runner cancellation and error isolation** (`src/zrb/runner/chat/chat_session_runner.py`): A `TimeoutError` racing with an external cancel no longer falls through as a silent success; the underlying LLM task is now explicitly cancelled and awaited before the shared task config is restored; a failed/timed-out request no longer kills the whole session loop, so queued messages keep being processed. The `interactive` shared-context default is now explicitly `"false"` so each SSE turn doesn't replay full history and re-fire session-start/end hooks.
- **Fix: deferred coroutine draining and exception loss** (`src/zrb/session/session.py`): `wait_deferred` now drains coroutines deferred *while* an existing batch is awaited, instead of gathering a single snapshot. Pruned finished coroutines now have their exception retrieved and logged instead of discarding it silently.
- **Fix: LLM history and tool-pairing correctness** (`src/zrb/llm/message.py`, `src/zrb/llm/task/history_mixin.py`, `src/zrb/llm/task/llm_task.py`): Merging sequential `ModelRequest`s now uses `dataclasses.replace` so fields like `instructions`/`run_id` survive the merge. `RetryPromptPart` is now treated as a tool return for orphan-stripping and pairing purposes, matching how providers actually serialize a failed tool call. Duplicate-user-message detection now compares only the most recent real user turn (skipping `[SYSTEM]`-prefixed bookkeeping turns) instead of scanning the whole history, so a recurring message like "continue" no longer gets mistaken for an already-sent turn. Toolsets are now resolved once per `_exec_action` call and threaded through to `_create_agent`, instead of being re-resolved (and re-entered/never-entered) separately.
- **Fix: `--attach` CLI flag silently ignored** (`src/zrb/builtin/llm/chat.py`): The `attach` input was accepted but never wired to the task, so attachments passed on the CLI were dropped. It's now split on commas and passed through as `attachment`.
- **Fix: `model_settings` silently ignored on `LLMChatTask`** (`src/zrb/llm/task/chat/exec_mixin.py`): The inner task now receives `self._model_settings`, instead of always falling back to `llm_config`'s.
- **Fix: `BoolInput` default string didn't match its own `<option>` values** (`src/zrb/input/bool_input.py`): `get_default_str` now returns `"true"`/`"false"` (lowercase) instead of `str(bool)`'s `"True"`/`"False"`, so the web UI `<select>` round-trips its own default.
- **Security: input rendering XSS** (`src/zrb/input/base_input.py`, `src/zrb/input/bool_input.py`, `src/zrb/input/float_input.py`, `src/zrb/input/int_input.py`, `src/zrb/input/option_input.py`, `src/zrb/input/password_input.py`, `src/zrb/input/text_input.py`): `to_html()` now HTML-escapes name/description/default before interpolating them into markup.
- **Security: dropped custom static-file route** (`src/zrb/runner/web_route/static/static_route.py`): Removed a hand-rolled `/static/{file_path:path}` handler that was already shadowed by the `StaticFiles` mount â€” dead code that would become a path-traversal hole the day route registration order changed. `StaticFiles` (with its own path containment) now fully owns `/static`.
- **Feature: `WEB_ENABLE_SECURE_COOKIES` config** (`src/zrb/config/mixins/web.py`, `src/zrb/config/web_auth_config.py`, `src/zrb/runner/web_util/cookie.py`): Auth cookies' `Secure` flag is now configurable (default `on`, preserving prior behavior) instead of hardcoded `True`, which silently broke cookie-based auth on plain-HTTP (non-localhost) deployments.

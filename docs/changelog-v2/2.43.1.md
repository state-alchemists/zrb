## 2.43.1 (June 27, 2026)

- **Change: only Shift+Tab cycles chat modes; plain Tab is freed (Termux keeps a Tab fallback)** (`src/zrb/llm/ui/default/keybindings_mixin.py`, `src/zrb/config/helper.py`, `src/zrb/config/mixins/foundation.py`, `src/zrb/llm/ui/base/commands_mixin.py`, `docs/adr/adr-0080.md`): Previously both Tab and Shift+Tab cycled the mode (normal → auto-accept-edits → plan) on every terminal, which made Tab unusable for anything else. Now Shift+Tab alone cycles the mode, matching Claude Code and leaving Tab free for completion. Termux is special-cased: there Tab and Shift+Tab emit the same byte (`0x09`), so Shift+Tab can't arrive — a new auto-detecting, env-overridable `CFG.IS_TERMUX` knob (set via `is_termux()`, true when `TERMUX_VERSION` is set or `PREFIX` contains `com.termux`) binds plain Tab to mode cycling there instead. Force the fallback on any terminal with `ZRB_IS_TERMUX=true`, or disable it with `ZRB_IS_TERMUX=false`. See ADR-0080.

- **Improvement: `zrb git changelog generate` feeds the LLM change statistics and handles the initial release** (`src/zrb/builtin/changelog.py`): The generator now passes `git diff --stat` (files changed, insertions, deletions) alongside the commit log so the model can gauge the scope of each change, and the embedded `git` tool now coaches a `--stat`-first strategy that skips generated/lock files (e.g. `poetry.lock`, `package-lock.json`). The initial release is handled correctly via git's empty-tree object, `--` disambiguation guards against refs that resemble paths, and git invocations return error strings instead of raising so a failed stat degrades gracefully to an empty section.

- **Improvement: file LLM tools attach `[SYSTEM SUGGESTION]` recovery hints on error** (`src/zrb/llm/tool/file_mv.py`, `src/zrb/llm/tool/file_rm.py`, `src/zrb/llm/tool/file_write.py`): Move/remove/write failures now return actionable guidance (check the path with List, verify permissions and disk space, pass `recursive=True`, etc.) following the project's error-handling convention, so the agent can self-correct instead of stalling.

- **Fix: mutable default arguments in `OptionInput` and sub-agent definitions** (`src/zrb/input/option_input.py`, `src/zrb/llm/agent/subagent/manager/manager.py`): Both used a shared mutable list (`options=[]`, `tools=[]`) as a default, so the list could leak across instances. They now default to `None` and allocate a fresh list per instance.

- **Maintenance: decompose god classes into focused modules and mixins** (`src/zrb/llm/agent/run/`, `src/zrb/llm/task/chat/`, `src/zrb/llm/lsp/`, `src/zrb/llm/ui/base/`, `test/config/`, `test/llm/ui/`): Large modules were split with no behavior change — the agent runner's setup, history, and permission/sandbox gate logic moved into `agent/run/setup.py`, `agent/run/history_utils.py`, and `agent/gates.py`; `LLMChatTask`'s execution path into `task/chat/exec_mixin.py`; the LSP server's operations into `lsp/server_operations.py`; and `BaseUI`'s accessors into `ui/base/properties_mixin.py`. The oversized `test_config.py` and `test_ui.py` suites were split by feature group. Several previously-lazy internal imports were also hoisted to module level where no longer needed lazily.

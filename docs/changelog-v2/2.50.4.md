## 2.50.4 (July 23, 2026)

- **Fix: sibling coroutines orphaned on first error** (`src/zrb/util/run.py`, `src/zrb/session/session.py`, `src/zrb/task/base/execution.py`, `src/zrb/task/base/lifecycle.py`): plain `asyncio.gather` propagates the first exception immediately and leaves the other coroutines running unretrieved. A new `gather_isolated` helper lets every sibling settle first, then re-raises the first exception (cancellation included) — no orphaned tasks, no "exception never retrieved" warnings. Task-chain, root-task, deferred-action, and deferred-monitoring fan-outs now use it.
- **Fix: a hanging readiness check could hang the whole run** (`src/zrb/task/base/execution.py`, `src/zrb/config/mixins/task_runtime.py`): the initial readiness wait now isolates its checks (`return_exceptions=True`) and honors a new optional `ZRB_TASK_READINESS_TIMEOUT` cap (milliseconds; `0` = off, the default, preserving existing behavior). Set a positive value to fail fast instead of blocking indefinitely on a check that never returns.
- **Observability: silent `except: pass` swallows replaced with debug logging** (across `src/zrb/llm/**`, `src/zrb/util/**`): best-effort `try/except` blocks that previously discarded the error now log it at `CFG.LOGGER.debug` (or carry an explanatory comment where logging isn't warranted), so failures in agent/hook loading, LSP queries, prompt composition, UI fan-out, and completion caches are diagnosable without changing control flow.
- **Refactor: voice engine readiness API** (`src/zrb/llm/voice/engine.py`, `src/zrb/llm/ui/default/keybindings_mixin.py`): `VoiceEngine` exposes `is_ready`, `is_vosk_model_ready()`, and `download_vosk_model()`, so the keybinding no longer reaches into module-level Vosk helpers or private state.

## 2.50.1 (July 11, 2026)

- **Session token usage in TUI status bar** (`src/zrb/llm/ui/base/ui.py`, `src/zrb/llm/ui/default/output_mixin.py`, `src/zrb/llm/util/stream_response.py`, `src/zrb/llm/agent/run/setup.py`): the TUI now tracks accumulated input and output tokens across all LLM requests in a session. The token totals (`💸 1.5k in · 34 out`) are displayed in the status bar alongside the "Ready" / "working" indicator, and reset when loading a different conversation history (`/load`). See [LLM Integration → Token Tracking](../advanced-topics/llm-integration.md#session-token-tracking).
- **Programming the prompt documentation** (`docs/advanced-topics/programming-the-prompt.md`): new page documenting the full ladder from a plain-string `message` up to a composed `PromptManager`, model-adaptive profiles, and seeding a chat with context from an upstream task. Replaces the inline dynamic-prompt section in `programming-the-agent.md`.
- **Test infrastructure: mock-based subprocess in runtime-shell tests** (`test/llm/tool/test_shell.py`): replaced real `create_subprocess_exec` calls with mocks so the Node.js runtime tests work on GitLab CI runners that don't ship `node`.
- **Test infrastructure: mock-based permission test** (`test/llm/tool/test_file_search.py`): replaced `chmod 0000` with a `_get_file_matches` mock so the unreadable-file test works when running as root.
- **`Env` default value** (`src/zrb/env/env.py`): `Env.__init__` now accepts `default=""` as a default, making `Env(name="x")` valid without an explicit default. Backward compatible.

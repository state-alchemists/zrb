## 2.50.0 (July 8, 2026)

- **Named style themes via `ZRB_THEME`** (`src/zrb/config/theme.py`, `src/zrb/config/mixins/theme.py`, ADR-0084): a single theme preset now supplies the default value for every style knob across the LLM chat TUI, the markdown renderer, and the CLI semantic colors. Ships `dark` (the historical defaults — a default install looks identical) and `light` (a dark-on-light palette). Any individually set `ZRB_LLM_UI_STYLE_*` / `ZRB_CLI_*` env still overrides the theme, and `zrb.config.theme.register_theme(name, values)` lets `zrb_init.py` add or override themes — registered values are merged onto `dark`, so a partial theme only lists the knobs it changes and inherits the rest. See `examples/themes/monokai/` for a worked example. An unknown `ZRB_THEME` logs a warning and falls back to `dark`.
- **Markdown rendering colors are now configurable** (`src/zrb/util/cli/markdown.py`, `src/zrb/config/mixins/llm_ui_styles.py`): the previously hardcoded Rich theme is driven by new `ZRB_LLM_UI_STYLE_MARKDOWN_{LINK,LINK_URL,H1,CODE}` knobs.
- **Previously hardcoded TUI styles lifted into config** (`src/zrb/llm/app/layout.py`, `src/zrb/llm/app/style.py`, `src/zrb/llm/ui/default/output_mixin.py`): the title-bar background and input-prompt color are now `ZRB_LLM_UI_STYLE_TITLE_BAR_BG` and `ZRB_LLM_UI_STYLE_PROMPT`; the jargon, choice-widget, and `shift+tab` hint shades follow the theme's `FAINT`/`TEXT` knobs. This is what makes the `light` theme coherent.
- **Install scripts: softer legacy `~/.local-venv` cleanup** (`install.sh`, `install.ps1`): instead of deleting `~/.local-venv` and rewriting the user's rc files/PowerShell profile, the installers now only uninstall `zrb` from the old venv (so the `pipx` copy wins on PATH) and print how to remove the directory. This avoids editing user shell config; the leftover activation block is existence-guarded and keeps activating the venv until the user removes the directory, so the installer says so rather than claiming it's gone. The `zrb`-uninstalled log now fires only when `zrb` was actually present.

## 2.50.2 (July 16, 2026)

- **Fix: unbounded tool payloads froze the TUI** (`src/zrb/llm/tool/web.py`, `src/zrb/llm/tool/mcp.py`, `src/zrb/llm/tool/code.py`): a tool return larger than the per-minute token budget could never be admitted by the rate limiter, livelocking the UI forever. Web page fetches, MCP tool results, and oversized single files in the repo-reader are now capped to `CFG.LLM_MAX_OUTPUT_CHARS` (keeping the head, where content is front-loaded) before becoming a message. MCP results are capped via pydantic-ai's `process_tool_call` hook so the toolset id and client stay intact for tool namespacing.
- **PDF fetching in `open_web_page`** (`src/zrb/llm/tool/web.py`): URLs are now read as PDFs when they end in `.pdf` (browser skipped entirely) or when the response `Content-Type` is `application/pdf` — including extensionless viewer URLs like `arxiv.org/pdf/1234.56789`, where the raw bytes are read from the same response with no extra round-trip. Text is extracted with `pdfplumber`.
- **Dependency: `pydantic-ai-slim[mcp]` bumped to `>=2.8.0,<2.9.0`** (`pyproject.toml`) — from `>=2.2.0,<2.4.0`. Required for the `process_tool_call` MCP hook above. Raises the supported floor; if you pin pydantic-ai below 2.8, update the pin.

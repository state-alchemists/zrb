## 2.48.0 (July 1, 2026)

- **Feature: `disallowedTools` frontmatter field for sub-agents** (`src/zrb/llm/agent/subagent/manager/`): New `disallowedTools` field in `.agent.md` frontmatter lets sub-agent definitions remove specific tools from their resolved toolset after the allowlist and factories have been applied. Accepts YAML list (`[Edit, Write]`) or comma-separated string (`Edit, Write`) formats.

- **Feature: Comma-separated `tools` frontmatter** (`src/zrb/llm/agent/subagent/manager/loader_mixin.py`): The `tools` field now accepts both YAML list and comma-separated string formats, matching the Claude Code sub-agent specification for broader compatibility.

- **Fix: Tool name resolution edge case** (`src/zrb/llm/agent/subagent/manager/manager.py`): Extracted tool name resolution into a dedicated `_resolve_tool_name()` helper that uses explicit `is not None` checks, preventing falsy values (e.g., empty string) from incorrectly falling through to `__name__`.

- **Docs: Sub-agent frontmatter spec** (`docs/advanced-topics/claude-compatibility.md`): Documented `tools` and `disallowedTools` formats, and their precedence.

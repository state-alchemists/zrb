## 2.49.2 (July 6, 2026)

- **Performance: defer `pydantic_ai` import off the `import zrb` path** (`src/zrb/llm/common_tools.py`, `src/zrb/builtin/llm/chat.py`, `src/zrb/llm/agent/subagent/manager/manager.py`, `src/zrb/llm/task/chat/exec_mixin.py`, `src/zrb/llm/task/llm_task.py`): Importing `zrb` eagerly pulled in `pydantic_ai` (~1.7s) because the `llm_chat` and `sub_agent_manager` singletons called `apply_common_tools` at module load, which transitively imports the LLM tool modules. New `defer_common_tools(host)` / `ensure_common_tools(host)` helpers register the tool surface lazily: the two singletons now defer, and each host (`LLMChatTask`, `LLMTask`, `SubAgentManager`) drains the deferred registration on its first run, so the heavy import lands on the first agent build instead of on every CLI invocation. `import zrb` drops from ~5.3s to ~0.4s. No public API change — `apply_common_tools` is untouched and the deferral state lives on a private attribute.
- **Docs: custom-tool registration and the deferred pattern** (`docs/advanced-topics/llm-integration.md`): Documented `add_tool_factory`, `defer_loading=True` for keeping rarely-used tools' schemas out of every turn, and recommended `defer_common_tools(host)` (over eager `apply_common_tools`) as best practice when equipping a custom `LLMTask`/`LLMChatTask` with zrb's shipped tool surface, since eager registration taxes every `zrb` command with the `pydantic_ai` import.

## 2.48.2 (July 2, 2026)

- **Security: SSH password passed via `SSHPASS` env var instead of command line** (`src/zrb/util/cmd/remote.py`, `src/zrb/task/cmd_task.py`, `src/zrb/task/rsync_task.py`): Replaced `sshpass -p <password>` with `sshpass -e` across all remote command and rsync paths. The password is now set in the `SSHPASS` environment variable and consumed by `sshpass` from that env var only — it never appears in the process list (`ps aux`) or shell command string. Removed the `password` parameter from `get_remote_cmd_script()`; callers set `SSHPASS` in the subprocess environment before invoking it.
- **Fix: `SafeToolsetWrapper` generic parameter** (`src/zrb/llm/agent/common.py`): Annotated `SafeToolsetWrapper` as `WrapperToolset[None]` to restore correct type resolution against pydantic-ai's type-safe toolset wrapper. Also pinned `deps_type=type(None)` on the `Agent()` constructor so the `toolsets` and `model_settings` parameters resolve against the correct overload instead of the `deps_type=object` default.
- **Fix: `_get_truncated_event_part_args` type annotation** (`src/zrb/llm/util/stream_response.py`): Widened the `event` parameter type to `AgentStreamEvent | ToolCallEvent` for compatibility with pydantic-ai ≥2.2.0, where `ToolCallEvent` replaced the narrower `FunctionToolCallEvent`.
- **Documentation: corrections and improvements across 30+ docs** (`docs/`): Fixed outdated references, improved clarity, and updated formatting across core concepts, task types, advanced topics, and configuration docs.

## 2.42.1 (June 25, 2026)

- **Fix: rename `{ACTIVE_SKILLS}` placeholder to `{PREACTIVATED_SKILLS}`** (`src/zrb/llm/prompt/claude.py`, `src/zrb/llm/prompt/markdown/mandate.md`, `src/zrb/llm/prompt/manager.py`, `src/zrb/config/mixins/llm_prompt.py`): The skill-catalogue placeholder for pre-activated skill content introduced in 2.42.0 was renamed from `{ACTIVE_SKILLS}` to `{PREACTIVATED_SKILLS}` to better reflect that these skills are loaded up front at session start. **Migration:** any custom `mandate.md` override (via `LLM_PROMPT_DIR` / `ZRB_LLM_PROMPT_MANDATE`) that still contains `{ACTIVE_SKILLS}` must be updated to `{PREACTIVATED_SKILLS}` — unmatched placeholders are left verbatim in the rendered system prompt with no warning.

- **Improvement: remove `README.md` / `AGENTS.md` from project-type markers** (`src/zrb/llm/prompt/system_context.py`): These files are not tech-stack indicators and the `"Docs"` / `"Agents"` labels they produced had no corresponding entry in `_PROJECT_TOOLS`, making them orphaned noise in the `- Project:` line. Their file paths continue to be surfaced by `create_project_context_prompt` in the `project_context` section, which already performs a richer ancestor-chain scan.

- **Improvement: add `docker-compose.yaml` to project-type markers** (`src/zrb/llm/prompt/system_context.py`): Docker Compose accepts both `.yml` and `.yaml` filenames; adding `docker-compose.yaml` ensures the `Docker Compose` label appears for projects that use the `.yaml` extension.

- **Improvement: add `glab` to detected utility tools** (`src/zrb/llm/prompt/system_context.py`): The GitLab CLI (`glab`) is now detected alongside `gh` and surfaced in the runtime tool list when present.

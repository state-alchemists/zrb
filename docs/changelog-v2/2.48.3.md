## 2.48.3 (July 2, 2026)

- **Fix: slash commands not resolved in interactive chat sessions** (`src/zrb/llm/task/chat/runner_mixin.py`): The interactive session path built its default UI kwargs before resolving custom/slash commands against the initial message, so a slash command passed as the starting message only worked in non-interactive runs. `_run_interactive_session` now resolves custom commands up front — mirroring `_run_non_interactive_session` — and passes the already-resolved list into `_build_default_ui_kwargs` instead of resolving it a second time.

## 2.50.3 (July 17, 2026)

- **Fix: context usage double-counting** (`src/zrb/llm/ui/base/ui.py`): `accumulate_usage` now reads `context_usage.input_tokens` directly instead of summing `input_tokens + cache_read_tokens + cache_write_tokens`. pydantic-ai's `AbstractUsage` contract already folds cache reads and writes into `input_tokens`, so the old formula counted the cache twice.
- **Fix: sync tools blocked the TUI event loop** (`src/zrb/llm/agent/common.py`): `create_safe_wrapper` is a coroutine function, so pydantic-ai never applies its own executor offload for sync tools — inline they froze the TUI for the tool's duration. Sync tools now run via `asyncio.to_thread`.
- **Fix: `ModelRetry` swallowed by safe wrapper** (`src/zrb/llm/agent/common.py`): `ModelRetry` exceptions are now re-raised so pydantic-ai's retry protocol works; previously they were caught and flattened into an opaque error string.
- **Fix: MCP `BinaryContent` (images) stringified and truncated** (`src/zrb/llm/tool/mcp.py`): `cap_mcp_result` now passes non-string, non-sequence, non-dict results through untouched. Only text fields are truncated — stringified `BinaryContent` (images) no longer lose their payload.
- **Fix: JSON truncation produced invalid JSON** (`src/zrb/llm/tool/code.py`): truncating the serialized string mid-byte left unterminated strings and missing braces. A new `_fit_file_payload` helper truncates the dominant text field *inside* the dict and re-serializes, so the extractor always receives valid JSON.
- **Fix: asyncio `StreamReader` 64 KB line limit** (`src/zrb/llm/tool/shell.py`): `readline()` raised on a single long line (minified JS, one-line JSON logs), losing all output. The `limit` parameter is now 8 MB. A post-start stream error also now terminates the subprocess instead of leaving it running detached.
- **Fix: terminal width always fell back to 80** (`src/zrb/llm/tool_call/argument_formatter/util.py`): the TUI redirects stdout/stderr to a capture pipe, so `shutil.get_terminal_size()` only saw the pipe and returned 80. The `format_diff` helper and the file-edit formatters now use `get_terminal_size()` which reads stdin (fd 0, not redirected) to recover the real width.
- **Fix: PDF text eaten by HTML converter** (`src/zrb/llm/tool/web.py`): `open_web_page` now skips the HTML→markdown converter for PDFs (their text is already plain). `<...>`-looking sequences (`#include <stdio.h>`, generics, emails) are no longer stripped as tags. The sync HTTP fallback and PDF parsing also run off-loop via `asyncio.to_thread`.
- **Fix: history corruption on crash** (`src/zrb/llm/history_manager/file_history_manager.py`): `_save_data_to_file` now writes to a sibling `.tmp` file and renames into place. A crash mid-write no longer truncates the live conversation file.
- **Performance: TUI event loop offloading** (`src/zrb/llm/prompt/live_context.py`, `src/zrb/llm/prompt/manager.py`, `src/zrb/llm/task/builder_mixin.py`, `src/zrb/llm/task/llm_task.py`, `src/zrb/llm/tool_call/handler.py`, `src/zrb/llm/tool_call/argument_formatter/*.py`): git subprocesses, history load/save, live-context rendering, argument formatting, and diff rendering now run via `asyncio.to_thread` so the per-turn hot path cannot freeze prompt_toolkit's event loop.
- **Fix: scroll-follow detection O(buffer) scan** (`src/zrb/llm/ui/default/output_mixin.py`): the cursor-on-last-line check used `document.cursor_position_row >= document.line_count - 1`, which rebuilds the Document's line index (an O(buffer) scan) on every streamed chunk. Replaced with an O(1) `str.find("\n", cursor) == -1` check on the raw string.
- **UI: status bar 2-row height** (`src/zrb/llm/app/layout.py`, `src/zrb/llm/ui/default/output_mixin.py`): status bar height raised from 1 to 2 rows; a leading newline separates the token-usage line from the hint text.

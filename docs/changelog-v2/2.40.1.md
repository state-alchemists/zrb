## 2.40.1 (June 23, 2026)

- **Feature: `EnvField.fallback` and `EnvField.transform`** (`src/zrb/config/env_field.py`): Two new descriptor parameters. `fallback` returns a fixed value when `cast` raises (e.g. `EnvField(int, fallback=0)` degrades a garbage env var to `0` instead of crashing). `transform` applies a `callable(value, host)` after casting, enabling post-read transformations that depend on sibling config (e.g. clamping a token threshold against `LLM_MAX_TOKEN_PER_MINUTE`).

- **Migration: all 15 `LLMContentMixin` properties to `EnvField`** (`src/zrb/config/mixins/llm_content.py`): Every config knob in `llm_content.py` is now an `EnvField` descriptor instead of a hand-written `@property`+setter pair. Five lenient-int fields (`LLM_HISTORY_BACKUP_RETAIN`, `LLM_HISTORY_SUMMARIZATION_WINDOW`, `LLM_HISTORY_MAX_DISPLAY_CHARS`, `LLM_HISTORY_TRUNCATE_LENGTH`, `LLM_FILE_READ_LINES`) use `fallback=0` to preserve the existing safe-degradation behavior. Five token-threshold fields (`LLM_CONVERSATIONAL_SUMMARIZATION_TOKEN_THRESHOLD`, `LLM_MESSAGE_SUMMARIZATION_TOKEN_THRESHOLD`, `LLM_REPO_ANALYSIS_EXTRACTION_TOKEN_THRESHOLD`, `LLM_REPO_ANALYSIS_SUMMARIZATION_TOKEN_THRESHOLD`, `LLM_FILE_ANALYSIS_TOKEN_THRESHOLD`) use `transform` for post-read clamping. Five simple fields (`LLM_HISTORY_DIR`, `LLM_SNAPSHOT_DIR`, `LLM_JOURNAL_DIR`, `LLM_JOURNAL_INDEX_FILE`, `LLM_ENABLE_REWIND`) were migrated in the previous commit. All 15 now appear in `zrb config explain`.

- **Fix: `LLM_SANDBOX_WRITABLE_PATHS` and `LLM_SANDBOX_DENY_READ_PATHS` display** (`src/zrb/config/mixins/llm_sandbox.py`): Added `serialize=colon_join` so these list-valued fields render as colon-delimited strings in `zrb config explain` instead of Python list repr.

- **Improvement: system prompt clarity** (`src/zrb/llm/prompt/markdown/mandate.md`, `journal_mandate.md`, `journal_reminder.md`, `AGENTS.md`): Skill Activation section clarified — activation timing, "when unsure" guidance, and skill authority over Working Loop steps. Journal protocol restructured as a decision table with explicit verification-before-logging. Journal reminder tightened to emphasize verified findings only. `AGENTS.md` ADR and Changelog sections condensed to reference external docs.

- **Improvement: `replace_in_file` error message** (`src/zrb/llm/tool/file_edit.py`): System suggestion now says "Do not retry with guessed text" to discourage blind retries when `old_text` is not found.

## 2.48.1 (July 1, 2026)

- **Fix: `CmdTask`/`run_command` stdout-stderr interleaving and `\r`-driven progress output** (`src/zrb/util/cmd/command.py`): Replaced two independently scheduled `readline()`-based reader tasks with a single loop that multiplexes stdout and stderr, reacting to whichever stream has data first. Two decoupled tasks could each drain a burst of already-buffered lines before yielding to the other, reordering interleaved output relative to when it was actually written; a shared loop removes that source of skew. Reading raw bytes instead of `readline()` also means `\r`-driven progress output (docker/apt-style progress bars with no trailing `\n`) is now shown live instead of buffered until a newline arrives.
- **Fix: unbounded memory growth on an unterminated line** (`src/zrb/util/cmd/command.py`): A line with no `\r`/`\n` at all is now force-flushed once it grows past `CFG.CMD_BUFFER_LIMIT`, restoring a memory safety valve equivalent to the old `readline()` limit (which the raw-byte read no longer enforces on its own).

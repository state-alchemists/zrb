## 🔥 Core Features That Make Us Jealous

### 🛠️ Self-Healing Pipelines

Imagine this: Your pipeline breaks. Instead of a sad red X, Zrb-Flow kicks in, analyzes the error, and **automatically patches your scripts**.

How? By integrating with Docker and Kubernetes APIs, Zrb-Flow:
- Captures runtime behavior and detects anomalies
- Uses intelligent inference to understand *what went wrong*
- Generates and applies fixes—without breaking your coffee
- Logs every change for full auditability

It's not just automation. It's **autonomous responsibility**.

### 🐳 Docker-Native by Design

Built from the ground up to speak Docker's native language:
- Smart container orchestration
- Image optimization suggestions
- Build cache intelligence
- Real-time stream aggregation

### ☸️ Kubernetes Power User Mode

Zrb-Flow doesn't just run with K8s—it *dominates*:
- In-cluster job scheduling
- Pod lifecycle management via smart agents
- Cross-namespace dependency resolution
- Rollback recommendations backed by anomaly detection

## 💻 CLI First, Always

No web UI to load. No browser tabs to manage. Zrb-Flow lives where you do:

```bash
# Initialize your first pipeline
zrb-flow init

# Run with auto-heal enabled
zrb-flow run --auto-heal

# Watch the magic
zrb-flow logs --live | grep "self-heal"
```

Because if you're reading this, you're already a command-line purist. We respect that.

## 🛠️ Installation (Because You're Ready)

**Requirements:**
- Docker (obviously)
- Kubernetes cluster (optional but recommended for the good stuff)
- Terminal with a pulse

**Get Zrb-Flow:**

```bash
# Via npm
npm install -g zrb-flow

# Via pip
pip install zrb-flow

# Binary download (macOS/Linux)
curl -sSL https://get.zrb.flow/install.sh | sh
```

Then run:

```bash
zrb-flow init --your-first-pipeline
```
